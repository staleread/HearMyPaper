# Traceability Matrix

Матриця простежуваності відображає зв'язок між вимогами системи та відповідними тестами.

## Матриця

| ID | Вимога | Опис | Тест |
| -- | ------ | ---- | ---- |
| FR-01 | Завантаження роботи | Студент може завантажити PDF-роботу | TC-01 Unit / Integration |
| FR-02 | Перегляд робіт | Викладач може переглядати призначені роботи | TC-02 Access Control Test |
| FR-03 | Шифрування | Система шифрує документи перед відправкою | TC-03 Security Test |
| FR-04 | Створення проєкту | Користувач може створювати проєкт (курс) | TC-04 Functional Test |
| FR-05 | Управління студентами | Куратор може додавати студентів до курсу | TC-05 Functional Test |
| FR-06 | Конвертація | Викладач може ініціювати конвертацію PDF в аудіо | TC-06 Integration Test |
| NFR-01 | Spike-навантаження | Система витримує різке зростання трафіку | TC-07 Load (Spike Test) |
| NFR-02 | Час відповіді | Середній час відповіді ≤ 300 мс | TC-08 Load Test |
| NFR-03 | Доступність | Доступність системи ≥ 99.9% | TC-09 Reliability Test |
| SEC-01 | Контроль доступу | Заборонено доступ між рівнями доступу | TC-10 Access Control |
| SEC-02 | Захист передачі | Дані передаються лише в зашифрованому вигляді | TC-11 Security Test |

## Висновок

Якість системи забезпечується комбінацією функціонального, навантажувального тестування та перевірки безпеки. Ключовий акцент — перевірка моделі доступу Bell–LaPadula та криптографічної безпеки.
